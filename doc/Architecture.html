The Bridge pattern is used to disconnect the abstraction (Camera) from implementation (CameraImpl). It is possible to change one or both of them in a transparent manner, and the client does not notice the change.
The CameraManager is the central object in the library. It determines which implementation is to be used and provides an unified interface for camera management.

The configuration tree is a graph data structure which holds nodes, identified by paths and allows to store a set of values.
Each node in the configuration tree has a value and may have subnodes. There's always at least one node in the tree - the root node, which does not hold any value and can't be accessed directly.
